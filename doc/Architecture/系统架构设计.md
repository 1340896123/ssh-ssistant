# SSH Assistant 系统架构设计

## 样式系统架构

### 架构概览
SSH Assistant 采用四层样式架构，从底层变量到顶层组件样式，实现了一套可维护、可扩展的设计系统。

```
┌─────────────────────────────────────────────────────┐
│ Layer 4: 组件样式 (Vue Component Styles)            │
│ - Tailwind Utility Classes                           │
│ - Custom Component CSS                              │
└─────────────────────────────────────────────────────┘
                    ↑
┌─────────────────────────────────────────────────────┐
│ Layer 3: 工具类 (Utility Classes)                   │
│ - src/style.css 第328-803行                         │
│ - .glass, .glow-border, .neon-text, .card 等        │
└─────────────────────────────────────────────────────┘
                    ↑
┌─────────────────────────────────────────────────────┐
│ Layer 2: Tailwind 配置                              │
│ - tailwind.config.js                                │
│ - CSS 变量映射 (colors, spacing, shadows 等)        │
└─────────────────────────────────────────────────────┘
                    ↑
┌─────────────────────────────────────────────────────┐
│ Layer 1: CSS 变量 (Design Tokens)                   │
│ - src/style.css 第10-151行                          │
│ - 颜色、间距、阴影、字体、动画等设计令牌              │
└─────────────────────────────────────────────────────┘
```

### 当前样式架构 (Arctic Glow - 北极光风格)

#### Layer 1: CSS 变量系统
**位置**: `src/style.css` 第10-151行

**变量分类**:
```css
/* 主题色彩 - Arctic Aurora */
--color-primary: #00D4FF        /* 冰蓝 */
--color-secondary: #A855F7      /* 极光紫 */
--color-accent: #00FFB3         /* 北极绿 */

/* 背景色彩 - Deep Arctic */
--bg-base: #030712
--bg-primary: #0A1628
--bg-secondary: #0F1E36
--bg-tertiary: #152642
--bg-elevated: #1A3152

/* 发光效果 - Aurora Glow */
--glow-primary: 0 0 20px rgba(0, 212, 255, 0.4)...
--glow-secondary: 0 0 20px rgba(168, 85, 247, 0.4)...

/* 字体系统 */
--font-sans: 'Outfit', 'Inter', sans-serif
--font-mono: 'JetBrains Mono', monospace
--font-display: 'Orbitron', 'Outfit', sans-serif
```

**特效系统**:
- **Aurora 动画**: 背景极光渐变动画 (30s 循环)
- **Glow 效果**: 多级发光 (subtle, primary, secondary, intense)
- **Glass 效果**: 毛玻璃背景 + 模糊滤镜
- **Grid Overlay**: 微妙网格叠加层

#### Layer 2: Tailwind 配置
**位置**: `tailwind.config.js`

**关键配置**:
```javascript
colors: {
  primary: 'var(--color-primary)',      // CSS 变量映射
  secondary: 'var(--color-secondary)',
  bg: {
    primary: 'var(--bg-primary)',
    secondary: 'var(--bg-secondary)',
  },
  // ... 完整的设计令牌映射
}
```

#### Layer 3: 工具类
**位置**: `src/style.css` 第328-803行

**核心工具类**:
| 类名 | 功能 | 依赖变量 |
|------|------|----------|
| `.glass` | 毛玻璃效果 | `--bg-glass`, `--border-subtle` |
| `.glow-border` | 发光边框 | `--glow-primary`, `--border-primary` |
| `.neon-text` | 霓虹文字 | `--color-primary-glow` |
| `.gradient-text` | 渐变文字 | `--color-primary/secondary/accent` |
| `.aurora-pulse` | 极光脉冲 | `@keyframes aurora-pulse` |
| `.card-glow` | 发光卡片 | `--shadow-glow-card` |

#### Layer 4: 组件样式
**影响组件**: 14个核心组件
- `TerminalView.vue`, `FileManager.vue`, `AIAssistant.vue`
- `ConnectionList.vue`, `SessionTabs.vue`, `SettingsModal.vue`
- 等其他组件...

**样式使用方式**:
1. Tailwind 工具类 (90%+ 使用率)
2. 自定义工具类 (glass, glow-border 等)
3. 组件内联样式 (动态值)

## 架构变更评估

### 变更目标
从 Arctic Glow (北极光风格) → 浅色简约风格

### 破坏性变更分析

#### 1. CSS 变量系统重构
**影响范围**: 全局 (100% 覆盖)

**需重写的变量类别**:
- ✅ **主题色彩**: Primary/Secondary/Accent (3组 × 5个变体 = 15个变量)
- ✅ **背景系统**: 6个背景色 (从深色 #0A1628 → 浅色 #F9FAFB)
- ✅ **文字色彩**: 5个文字色 (从浅色 #E8F4FC → 深色 #111827)
- ✅ **边框色彩**: 5个边框色 (透明度调整)
- ❌ **发光效果**: 完全移除 (5个变量)
- ❌ **特殊背景**: --bg-glass 移除
- ✅ **字体系统**: 保持不变 (JetBrains Mono + Inter/Outfit)
- ✅ **间距/圆角/阴影**: 保持不变

**兼容性影响**:
- ⚠️ **无向后兼容需求** - 整体视觉风格替换
- ⚠️ **所有组件需视觉验证** - 颜色对比度可能失效

#### 2. 工具类清理
**需移除的工具类**:
```css
/* 完全移除 */
.glass, .glass-intense              /* 毛玻璃效果 */
.glow-border, .glow-border-*        /* 发光边框 (3个变体) */
.neon-text, .neon-text-*            /* 霓虹文字 (3个变体) */
.aurora-pulse, .glow-pulse          /* 脉冲动画 */
.gradient-text, .gradient-text-alt  /* 渐变文字 */
.card-glow                          /* 发光卡片 */
```

**需保留的工具类**:
```css
/* 保留并调整颜色 */
.card, .btn, .input, .panel, .tab
.badge, .modal-content, .tooltip
```

**新增工具类** (浅色风格需要):
```css
/* 建议新增 */
.surface-elevated     /* 浅色浮起效果 */
.border-subtle        /* 微妙边框 */
.shadow-soft          /* 柔和阴影 */
```

#### 3. 动画系统变更
**移除的动画**:
- `aurora-shift` - 背景极光动画
- `aurora-pulse` - 极光脉冲
- `glow-pulse` - 发光脉冲
- `shimmer` - 闪烁效果

**保留的动画**:
- `fade-in-up`, `scale-in`, `slide-in-right` - 基础过渡动画
- Tailwind 配置中的 `fade-in`, `scale-in`, `slide-up` 等

#### 4. 全局样式变更
**移除**:
- `body::before` - Aurora 背景渐变
- `body::after` - Grid 网格叠加
- 所有发光相关的 text-shadow

**调整**:
- `::-webkit-scrollbar` - 滚动条样式 (浅色适配)
- `::selection` - 选中文字颜色 (对比度调整)

### 性能影响评估

#### 性能提升 ✅
1. **移除动画**: 删除 4 个复杂 CSS 动画
   - Aurora 背景动画 (30s 循环)
   - 脉冲动画 (持续执行)
   - **预期收益**: 减少 GPU 渲染压力 ~15-20%

2. **简化阴影**: 从多层发光阴影 → 单层标准阴影
   - `box-shadow` 复杂度降低
   - **预期收益**: 重绘性能提升 ~10%

3. **移除模糊滤镜**: `.glass` 类移除
   - `backdrop-filter: blur()` 是性能杀手
   - **预期收益**: 模糊渲染开销降低 ~25%

#### 无性能影响
- CSS 变量读取性能 (保持不变)
- Tailwind 编译后体积 (相当)

## 风险评估

### 风险等级: 🟢 低风险

### 风险因素

#### 1. 视觉一致性风险 🟡 中
- **风险**: 组件样式可能需要逐个调整
- **缓解**: CSS 变量映射确保 90%+ 自动适配
- **验证**: 需要完整视觉回归测试

#### 2. 可访问性风险 🟢 低
- **风险**: 浅色背景对比度可能不足
- **缓解**: 使用 WCAG AA 标准 (4.5:1 对比度)
- **验证**: 自动化对比度检查工具

#### 3. 依赖风险 🟢 极低
- **风险**: 第三方组件库兼容性
- **缓解**: 仅使用 Tailwind + 自定义 CSS
- **验证**: 无外部 UI 库依赖

#### 4. 性能风险 🟢 极低
- **风险**: 样式变更影响渲染性能
- **缓解**: 移除动画和模糊滤镜反而提升性能
- **验证**: Lighthouse 性能分数应提升

## 实施策略

### 推荐方案: 自底向上渐进式替换

```
Phase 1: CSS 变量重写 (基础层)
   ↓
Phase 2: 工具类清理 (中间层)
   ↓
Phase 3: 组件样式验证 (应用层)
   ↓
Phase 4: 视觉优化与微调 (打磨)
```

### Phase 1: CSS 变量重写
**目标**: 建立新的设计令牌系统

**任务**:
1. 重写 `src/style.css` 第10-151行 CSS 变量
2. 移除所有 glow 相关变量
3. 更新颜色值为浅色系
4. 移除 `body::before` 和 `body::after`

**验证**: 变量替换后编译无错误

### Phase 2: 工具类清理
**目标**: 移除不兼容的工具类

**任务**:
1. 删除 `.glass`, `.glow-border`, `.neon-text` 等类
2. 保留 `.card`, `.btn`, `.input` 等基础类
3. 添加新的浅色风格工具类 (如需要)

**验证**: 工具类引用处无报错

### Phase 3: 组件样式验证
**目标**: 确保所有组件视觉正常

**任务**:
1. 逐个验证 14 个核心组件
2. 检查颜色对比度
3. 调整不适配的样式
4. 测试交互状态 (hover, active, disabled)

**验证**: 组件功能正常, 视觉符合预期

### Phase 4: 视觉优化
**目标**: 打磨细节, 提升品质

**任务**:
1. 微调间距和圆角
2. 优化阴影层级
3. 统一交互反馈
4. 性能测试

**验证**: Lighthouse 分数 > 90

## 架构决策记录 (ADR)

### ADR-001: 移除所有发光特效
**状态**: ✅ 已批准

**理由**:
- 浅色风格不需要发光特效
- 发光特效在浅色背景下视觉混乱
- 移除可显著提升性能

**影响**:
- CSS 变量删除 5 个
- 工具类删除 8 个
- 动画删除 4 个

### ADR-002: 移除毛玻璃效果
**状态**: ✅ 已批准

**理由**:
- `backdrop-filter` 性能开销大
- 浅色风格下效果不明显
- 简化代码复杂度

**影响**:
- 工具类删除 2 个 (`.glass`, `.glass-intense`)

### ADR-003: 保持语义化命名
**状态**: ✅ 已批准

**理由**:
- `--color-primary` 等命名已语义化
- 便于未来主题切换
- 降低维护成本

**影响**:
- 变量名保持不变
- 仅变更变量值
